Detailed design is a phase in the system development lifecycle 	where the high-level design
of the Fashionista system is converted into detailed and implementable specifications. 	In
this phase, 	components such as system architecture, 	database structure, 	algorithms, 	user
interfaces, 	and modules are defined in detail. 	The main goal is to create a clear blueprint
that developers can follow 	to build the system accurately. 	It includes diagrams,
pseudocode, 	and technical documentation to ensure the system is efficient, 	feasible, 	and
easy 	to maintain.

6. 	TESTING
Introduction:
Testing is the process of checking 	whether the Fashionista 	website is 	working
correctly 	and meeting the required conditions. 	It helps to identify 	errors, 	bugs, 	or
differences between expected and actual results. 	In our project, 	software testing is
used to ensure that all features like login, 	product browsing, 	cart, 	and order
placement 	work properly 	under both normal and abnormal conditions. 	Testing
mainly 	focuses on detecting errors and improving the quality 	of the system.
Software testing has three main purposes:
 	Verification: 	Ensures that the system meets technical requirements such as
correct login 	validation, 	proper database storage, 	and smooth functionality
of modules.
 	Validation: 	Ensures that the system satisfies user needs like easy 	shopping
experience, 	proper product display, 	and simple navigation.
 	Defect Detection: 	Identifies differences between expected output and actual
output. 	These defects may 	occur due to errors in design, 	coding, 	or
database handling.
Two types of software testing:
1. 	Black Box 	Testing:
Black box 	testing is performed 	without knowing the internal code of the Fashionista
system.
In this method, 	we test the application by 	giving inputs and checking outputs.
For example:
 	Entering login details and checking 	whether login is successful
 	Adding products to cart and 	verifying correct items are displayed
 	Placing an order and checking order confirmation
This testing ensures that the system 	works correctly 	from the user’s point of 	view.
2. 	White Box 	Testing:
White box 	testing is done 	with knowledge of internal code and logic.
In Fashionista, 	this includes:
 	Checking PHP code for login 	validation
 	Verifying database queries

 	Ensuring proper 	working of backend logic like cart and order processing
This testing helps in checking code correctness and improving performance.

a. 	Unit Testing
Unit tests are 	written from a programmer’s perspective. 	They 	ensure that a particular
method of a class successfully 	performs a set of specific tasks. 	Each test confirms that a
method produces the expected output 	when given a known input. 	Each test examines an
individual component that is new 	or has been modified. 	Unit tests focus on functionality
and reliability. 	Unit testing is done in a test environment prior to system integration. 	If a
defect is discovered at unit test, 	the severity 	of the defect 	will dictate 	whether or not it 	will
be fixed before the module is approved.
